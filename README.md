# Swiggy Sales Analysis (Python Project)

## Project Overview
This project analyzes Swiggy food delivery sales data using Python to uncover key insights about sales performance, customer preferences, and city-level revenue trends.

The analysis was conducted using Python within a Jupyter Notebook environment, focusing on data cleaning, exploratory data analysis, and visualization.

## Tools & Technologies Used

- Python
- Pandas
- NumPy
- Matplotlib
- Plotly
- Jupyter Notebook

## Dataset
The dataset contains information related to food delivery orders including:

- Order Date
- City
- Food Category
- Price (INR)
- Rating
- Payment Method

## Project Objectives

The main goals of this analysis were:

- Understand overall sales performance
- Identify top performing cities
- Analyze weekly and quarterly sales trends
- Compare Veg vs Non-Veg revenue contribution
- Identify patterns in customer ratings

## Key Analysis Performed

### 1. Weekly Sales Trend
Analyzed sales performance across weeks to identify patterns in demand.

### 2. Quarterly Performance Analysis
Examined total sales, average rating, and order count by quarter.

### 3. Revenue by Food Category
Compared revenue contribution between Veg and Non-Veg food categories.

### 4. Top 5 Cities by Sales
Identified cities generating the highest revenue.

### 5. Customer Rating Analysis
Analyzed how ratings vary across orders.

## Key Insights

- Some cities contribute significantly more revenue than others.
- Non-Veg food categories generate a larger share of sales.
- Weekly sales trends reveal fluctuations in customer demand.
- High-rated orders often correlate with popular food categories.

## Project Output

The analysis provides actionable insights that could help food delivery platforms understand:

- Market demand
- Regional performance
- Customer preferences
